SkillPilot

面向 Agent 的 Agent 系统：
定向 Skill 搜寻、推荐与构造。

Claude 扩展安装的痛点

🤔类型选择困难

同一个需求可能落在 Skill、MCP Server、Claude Code Plugin 或混合方案之间，安装前必须先判断任务到底需要说明型能力、外部连接还是完整工作流。

😣扩展源众多

官方仓库、社区市场、GitHub 仓库和网页数据库同时存在，入口分散、标准不一，搜索时很难判断应该从哪里开始。

😭难确定是否合适

很多 SKILL.md 说明冗长，能力边界并不直观；Plugin 这类集成度更高的扩展，还需要进一步判断功能范围、依赖和潜在风险。

SkillPilot 多智能体系统

输入

  • 一段自然语言需求，即想让 Agent 扩展的能力。

输出

  • 需求理解、搜索源、候选评分和最终决策。
  • JSON 决策轨迹：完整保留 agent/skill trace。
  • 若没有符合需求的现存 Skill，则帮助用户主动构建一个 Skill。

Candidate Evaluation & Decision Agents

从原文证据中评估候选，再根据阈值决定推荐还是构造 Skill

Candidate Evaluation Agent

Input

ParsedRequirement TypeClassification RetrievedContent[]
CandidateEvaluator

让 LLM 直接阅读 RetrievedContent.content

生成候选字段、匹配能力、缺失能力、风险原因

Output

CandidateEvaluation[] Candidate capability_score documentation_score safety_score type_score match_score
match = 0.45 capability + 0.15 type + 0.20 docs + 0.20 safety
按 match_score 排序后传递

Decision Agent

无候选 / 最高分 < 0.45 → build_custom_skill
最高候选 risk_level = high → build_custom_skill
最高分 ≥ 0.75 且风险可接受 → recommend_existing
中等相关但能力有缺口 → recommend_with_custom_extension

总结与展望

SkillPilot 帮助用户在复杂扩展生态里做更可解释、更安全的信息决策。

目前成果

  • 搭建了 pipeline-agent-skill 多智能体工作流
  • 实现需求解析、类型判断、搜索规划、候选读取与评分
  • 支持 Claude Skill / Plugin / MCP / mixed 类型决策
  • 引入 LLM 结构化判断与确定性安全 guardrail
  • 可生成 Markdown 推荐报告与 JSON 决策轨迹
  • 可在无合适候选时生成自定义 Skill 草案
  • 提供 CLI 与本地 Web UI 两种交互方式

改进方向

  • 扩展更多 source-specific searcher 与 reader
  • 提升 marketplace / registry 检索质量
  • 增强候选评分的证据引用与可验证性
  • 细化高风险权限、token、写操作和命令执行的安全策略
  • 提升整个多智能体系统的协作效率，降本增效
